# Persistent Document State v1 Summary

**Milestone:** v0.11
**Status:** Implemented and verified; closed pending owner commit and tag

## Milestone Purpose

v0.11 establishes a persistence-neutral operational state boundary for Document Intelligence. It defines how document lifecycle and processing read state can be recorded safely and read through the existing Workflow Query Facade without allowing API or UI consumers to depend on repositories directly.

The milestone proves the boundary with deterministic process-local repositories. It does not provide durable production persistence.

## Delivered Capabilities

- Public persistence-neutral package at `src/document_state/`.
- Ten immutable, JSON-compatible record families covering documents, lifecycle events, processing snapshots, validation issues, matching summaries, review references, correction summaries, reprocess plans, workflow runs, and audit events.
- Fixed statuses, filters, deterministic ordering declarations, and bounded limit/offset pagination.
- Privacy validators rejecting raw document content, rows, correction values, artifact payloads, stack traces, storage details, unsafe keys, and unbounded metadata.
- Stable repository errors: `invalid_record`, `invalid_query`, `not_found`, `conflict`, `source_unavailable`, and `internal_error`.
- Separate structural read and write repository `Protocol` surfaces.
- Deterministic lock-protected in-memory repositories with immutable returns and defensive contract reconstruction.
- Optimistic `expected_version` checks for mutable snapshots.
- Stable-key idempotency for append-only records, with safe conflict behavior.
- Read-only `DocumentStateQueryFacadeAdapter` mapping repository records into existing v0.10 read models.
- Recursive boundary, privacy, error, repository, API/UI isolation, and full-regression verification.

## Final Architecture

```text
Document Intelligence API
        |
        v
FacadeDocumentIntelligenceProvider
        |
        v
Workflow Query Facade
        |
        v
DocumentStateQueryFacadeAdapter
        |
        v
Document State repositories
        |
        v
In-memory deterministic store
```

This chain is the approved repository-backed read composition and is verified through injected ports. Production source selection remains deferred; API and Streamlit do not receive repository objects or import Document State directly.

## Phase Summary

1. **Contracts and interfaces:** Added immutable records, privacy rules, pagination, safe errors, deterministic filters/orderings, and separated read/write ports.
2. **In-memory repositories:** Added deterministic process-local implementations with operation locking, defensive reconstruction, optimistic versions, and append idempotency.
3. **Query Facade adapter:** Added explicit read-only mappings from Document State records to public Workflow Query Facade read models.
4. **Boundary and privacy hardening:** Added recursive dependency checks, API/UI isolation checks, repository-to-facade privacy verification, tamper resistance, safe error mapping, and conflict tests.
5. **Release closure:** Re-ran focused and full regressions and completed summary, handoff, release, roadmap, debt, plan, and ADR updates.

## Runtime Boundaries

- API and Streamlit must read through the Document Intelligence API and Workflow Query Facade, never repositories.
- Core `src/document_state/` imports only the standard library and package-local modules.
- Core Document State must not import API, UI, Workflow Runtime implementations, runtime internals, storage, telemetry, database, external-service, FlowSync, or competitor-price modules.
- The explicit adapter may import only public Document State and public Workflow Query Facade boundaries.
- Future database implementations remain behind existing repository interfaces.
- Future upload and processing producers write through Document State write ports rather than concrete repositories.

## Privacy And Safety

Query-facing records contain bounded summaries rather than source payloads. Correction records never contain raw old or new values. Audit metadata is scalar, bounded, and allowlisted. Repository and facade errors expose stable codes and safe messages rather than IDs, payloads, exceptions, or stack traces.

## Backward Compatibility

v0.11 is additive. It does not alter v0.9 API contracts, v0.10 Query Facade read models, Streamlit behavior, root `dashboard.py`, legacy `src/api/app.py`, or competitor-price modules.
