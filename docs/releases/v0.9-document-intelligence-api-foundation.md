# v0.9 Document Intelligence API Foundation

## Release Name

Document Intelligence API Foundation v1

## Capabilities Added

- Separate versioned FastAPI application for Document Intelligence.
- Health, status, document, processing, validation, matching, review, correction-history, reprocess-plan, workflow-run, and audit-event GET endpoints.
- Strict success/error envelopes, API versioning, request IDs, and pagination metadata.
- Deterministic API-owned local provider with stable ordering and defensive copies.
- Streamlit `local_preview` and optional GET-only `api_preview` modes.
- Privacy-safe query errors and `404`, `405`, and global `500` responses.
- Request-ID sanitization/generation and header/body propagation.
- No-store, referrer, content-type, and frame security headers.
- Disabled-by-default CORS and GET-only OpenAPI verification.

## Compatibility Notes

This release is additive. Legacy `src/api/app.py`, root `dashboard.py`, competitor-price FlowSync, and backend runtime contracts are unchanged. Streamlit local preview remains the default.

The API contains deterministic preview data only. It has no direct runtime imports and is not a source of truth. Future live reads must use a public Workflow-owned query facade.

## Known Non-Goals

- Authentication, authorization, tenant isolation, or rate limiting.
- Production CORS, gateway, TLS, trusted hosts, or public deployment.
- Database, durable persistence, live runtime data, or Workflow query facade.
- POST, PUT, PATCH, DELETE, upload, review decision, correction, reprocessing, or workflow execution endpoints.
- Production telemetry or TestClient dependency alignment.
- FlowSync Document Intelligence UI.
- OCR, LLM processing, or external services.
