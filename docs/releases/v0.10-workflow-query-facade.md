# v0.10 Workflow Query Facade

## Release Name

Workflow Query Facade v1

## Capabilities Added

- Workflow-owned public read boundary under `src/workflow_runtime/query_facade/`.
- Immutable JSON-compatible read models across all Document Intelligence query areas.
- Bounded pagination, safe fixed filters, and deterministic ordering.
- Stable privacy-safe query errors and read-only structural ports.
- Deterministic in-memory facade provider.
- API facade-provider adapter preserving v0.9 provider shapes.
- Safe facade error mapping for invalid, missing, unavailable, and internal outcomes.
- Recursive import, privacy, API contract, GET-only method, request-ID, and security-header verification.

## Architecture

```text
Document Intelligence API
  -> FacadeDocumentIntelligenceProvider
  -> Workflow Query Facade Port
  -> InMemoryWorkflowQueryFacade
  -> Immutable read models
```

## Compatibility Notes

This release is additive. v0.9 paths, GET-only methods, successful payload meanings, envelopes, pagination, request IDs, and security headers remain unchanged. Streamlit `api_preview` remains compatible. Legacy API, dashboard, and competitor-price code remain separate and unchanged.

The current provider is deterministic preview infrastructure. It does not connect live runtime state or establish production readiness.

- Production CORS, gateway, TLS, deployment policy, or telemetry.
