# v0.19 Upload + Processing Activation

## Release Status

Implemented, verified, and closed pending owner tag.

## Capabilities Added

- Upload Runtime contracts, deterministic validation, command/result/error models, idempotency, and staging port contract.
- Guarded JSON metadata upload API with tenant and permission enforcement.
- Controlled activation service over a validated command and opaque staged-artifact reference.
- Safe ingestion activation and Document State received-write intents through injected ports.
- Immutable processing status/progress models, projections, tenant-scoped queries, and safe failure summaries.
- Upload collection/detail, progress, timeline, and document processing-status API routes.
- FlowSync `/uploads` page with browser-local metadata selection and explicit validation preview.
- Recent uploads, API-supplied processing timeline, approximate-progress labeling, manual refresh, and document processing-status panel.
- Source validation proving no raw-file transmission implementation exists.

## Architecture

```text
FlowSync upload preview
  -> Guarded upload metadata API
  -> Upload Runtime validation
  -> Opaque staging boundary
  -> Activation intents
  -> Ingestion / Document State adapter ports
  -> Progress projections
  -> FlowSync timeline
```

The staging and adapter-port portion is a contract boundary only. No real artifact staging or ingestion execution is enabled.

## API Contracts And Mutation State

Current routes:

- `POST /api/v1/documents/upload`
- `GET /api/v1/uploads`
- `GET /api/v1/uploads/{upload_id}`
- `GET /api/v1/uploads/{upload_id}/progress`
- `GET /api/v1/uploads/{upload_id}/timeline`
- `GET /api/v1/documents/{document_id}/processing-status`

The POST is JSON metadata only. It accepts no multipart body or raw bytes, stages nothing, and executes no ingestion. Valid authorized metadata returns the governed staging-disabled state; disabled staging returns `upload_staging_not_enabled`. No upload path triggers export or ERP.

## Compatibility And Safety

The v0.17 visual identity and existing routes remain intact. FlowSync sends only filename, size, extension, and browser content type after explicit action. It makes no client-side authorization decision and exposes no tenant/actor controls. Public surfaces exclude raw content, paths, credentials, tokens, claims, stack traces, raw exceptions, unrestricted metadata, and fabricated progress.
