# v0.17 FlowSync Document Intelligence UI

## Release Name

FlowSync Document Intelligence UI v1

## Capabilities Added

- Isolated Vite, React, and TypeScript product application.
- Responsive sidebar/header shell and stable route catalog.
- Strict GET-only API client with standard-envelope validation and safe errors.
- Read-only document list/detail, validation, matching, review, workflow, and audit views.
- API-authoritative unauthorized, forbidden, concealed-not-found, unavailable, and malformed-response states.
- Safe runtime preview and no-API startup behavior.
- Semantic operational presentation, keyboard skip navigation, and mobile menu accessibility basics.
- Lockfile-controlled advisory-free dependency graph and generated-output tracking guards.

## Architecture

```text
FlowSync Document Intelligence UI
  -> GET-only API client
  -> Document Intelligence API
  -> Platform Runtime
  -> Document State / Lifecycle / Query Facade
```

The UI imports no backend Python, Streamlit, or competitor-price modules. It does not enforce permissions or tenant scope and contains no mutation actions.

## Route Coverage

`/documents`, `/documents/:documentId`, `/documents/:documentId/validation`, `/documents/:documentId/matching`, `/review`, `/review/:reviewCaseId`, `/workflows`, `/audit`, `/settings/runtime-preview`, `/unauthorized`, and `/unavailable`.

## Compatibility Notes

- Existing API paths, methods, payload meanings, envelopes, request IDs, authorization behavior, and runtime composition are unchanged.
- Streamlit, root `dashboard.py`, legacy `src/api/app.py`, and Competitor Price remain separate and unchanged.
- The application is read-only and requires the API to run separately for live data.

## Known Non-Goals

- Real login/session/identity-provider integration.
- Upload, correction submission, review decisions, reprocessing, workflow execution, export, or other mutations.
- Protected document preview or protected correction values.
- Production hosting, deployment security, telemetry, analytics, or E2E CI.
- Full screen-reader/contrast audit, full viewport matrix, final theme, or pixel-perfect polish.
- ERP/export adapters and UI actions.
