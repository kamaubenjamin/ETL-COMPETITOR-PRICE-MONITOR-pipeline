# v0.8 Document Intelligence Operator Console

## Release Name

Document Intelligence Operator Console - Streamlit v1

## Capabilities Added

- Separate Streamlit console for the Intelligent Document Processing Platform.
- Sidebar workspace, document, workflow, runtime, and review filters.
- Overview metrics for received, processed, review-required, failed, and export-ready documents.
- Inbox, processing, validation, matching, review queue, workflow run, and audit views.
- Disabled upload placeholder with no persistence or backend behavior.
- Deterministic local mock datasets covering document and review lifecycles.
- Defensive local provider methods for every console dataset and summary metrics.
- Pure view-model shaping and reusable Streamlit display components.
- Deterministic filtering and copy-isolation test coverage.
- Read-only Review Runtime preview built from public case, correction, decision, audit, and dry-run reprocess contracts.
- Review Queue correction counts, decision/reason codes, assignments, statuses, and reprocess state.
- Review Runtime audit events projected without correction values or raw artifact payloads.
- Numbered task navigation, grouped operational overview, and consistent section context.
- Display-only lifecycle and priority labels plus explicit filtered empty states.
- Prominent local-preview run mode and disabled-upload safety messaging.

## Compatibility Notes

The legacy competitor-price `dashboard.py` is unchanged. The new console does not import competitor-price modules or modify Review Runtime, Workflow Runtime, or other backend contracts.

Streamlit remains the internal/operator console. FlowSync Competitor Price remains separate, and a future FlowSync Document Intelligence surface requires its own milestone and contracts.

## Known Non-Goals

No live Review Runtime connection, API, database, external service, authentication, state mutation, real upload, OCR, LLM processing, or production deployment is included. All displayed records are deterministic local fixtures or in-memory contract samples.

## Post-Release Integration Note

v0.9 Phase 3 later adds an optional GET-only Document Intelligence API preview adapter. The original v0.8 `local_preview` behavior remains the default, and this does not retroactively make v0.8 authenticated, persistent, mutation-capable, or production-connected.
