# v0.15 Auth, Tenant, And Permission Boundaries

## Release Name

Auth, Tenant, And Permission Model v1

## Capabilities Added

- Provider-neutral immutable security contracts and safe decisions/errors.
- Fixed role and permission catalogs with deterministic default-deny policy.
- Explicit user, anonymous, service-account, and system principals.
- Deterministic local identity provider for development and tests.
- Authorization requests and reusable `PermissionGuard`.
- Tenant-aware `DocumentRecord`, tenant-scoped repositories, migration `002`, and Query Facade filters.
- Opt-in API auth modes and centralized guards for existing GET routes.
- Safe `401`, `403`, concealed `404`, and provider-unavailable responses.
- Streamlit `api_preview` local-demo identity controls with API-authoritative enforcement.

## Architecture

```text
Request / local demo identity
  -> IdentityProvider
  -> Principal
  -> AuthorizationContext
  -> AuthorizationRequest
  -> PermissionGuard
  -> Authorized tenant scope
  -> API provider / Query Facade tenant filter
  -> Document State repository narrowing
  -> Safe response / Streamlit preview
```

## Compatibility Notes

- Existing v0.9 paths, GET-only methods, payload meanings, envelopes, request IDs, pagination, and security headers are unchanged.
- Disabled auth remains the default local compatibility mode.
- Streamlit `local_preview` remains unchanged and default.
- No public tenant fields or mutation endpoints were added.
- Legacy documents are deterministically backfilled to `tenant-local` by migration `002`.

## Privacy And Safety

No tokens, credentials, raw claims, raw documents, raw rows, correction values, artifact payloads, storage paths, stack traces, or raw exception messages are exposed through security decisions, API responses, or Streamlit preview state.

## Known Non-Goals

- Real identity providers, token verification, sessions, MFA, or SSO.
- Supabase/Auth0/OAuth adapters, PostgreSQL, or row-level security.
- Public mutation/upload endpoints or writer authorization enforcement.
- Verified audit actor attribution persistence.
- Child-record tenant columns and cross-record tenant validation.
- Tenant/user administration, FlowSync authentication, or ERP/export enforcement.
- Production composition activation.
