# v0.7 Review / Correction Runtime

## Release Name

Review / Correction Runtime v1

## Capabilities Added

- Versioned JSON-compatible review, correction, decision, audit, request, and reprocess-plan contracts.
- Deterministic review lifecycle with eight states and five reviewer decisions.
- Idempotent case creation, assignment, filtering, optimistic version checks, and terminal resolution.
- Field-level controlled corrections with artifact and stage lineage.
- Ordered append-only audit events and failure-atomic in-memory operations.
- Declarative dry-run reprocess planning with deterministic artifact lists and privacy-safe audit metadata.
- Focused privacy, compatibility, repository, service, reprocess, and runtime-boundary coverage.

## Compatibility Notes

Legacy `pending` status input maps to `review_required`. Existing prototype modules remain available; canonical v1 operations use `src/review_runtime/contracts`, `repositories`, `services`, and `reprocess`. Review Runtime does not execute workflows or alter upstream runtime contracts.

## Known Non-Goals

- UI, API, Streamlit, FlowSync, database persistence, and external services.
- Authentication, authorization, reviewer workforce management, and notifications.
- Workflow review-stage implementation, acknowledgement handling, or real reprocess execution.
- OCR, LLM decisions, automated corrections, or direct domain-artifact mutation.
- Production durability or multi-process concurrency guarantees.
