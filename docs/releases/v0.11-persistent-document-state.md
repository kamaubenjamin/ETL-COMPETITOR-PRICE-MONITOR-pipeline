# v0.11 Persistent Document State / Repository Layer

## Release Name

Persistent Document State v1

## Capabilities Added

- Persistence-neutral `src/document_state/` package.
- Immutable JSON-compatible operational state records across ten record families.
- Privacy validators, fixed filters/orderings, bounded pagination, and stable safe errors.
- Separate read-only and write-only structural repository ports.
- Deterministic lock-protected in-memory repositories.
- Optimistic version checks for mutable snapshots.
- Stable-key idempotency and conflict behavior for append-only records.
- Read-only Document State to Workflow Query Facade adapter.
- Recursive import, API/UI isolation, privacy projection, repository safety, and error-mapping verification.

## Architecture

```text
Document Intelligence API
  -> FacadeDocumentIntelligenceProvider
  -> Workflow Query Facade
  -> DocumentStateQueryFacadeAdapter
  -> Document State repositories
  -> In-memory deterministic store
```

The repository-backed chain is verified through injected ports. Production composition and durable persistence are not included.

## Compatibility Notes

This release is additive. v0.9 API paths, GET-only methods, successful payload meanings, envelopes, pagination, request IDs, and security headers remain unchanged. v0.10 read models and Streamlit behavior remain compatible. Legacy API, dashboard, and competitor-price modules remain separate and unchanged.

## Known Non-Goals

- Durable database implementation, schema, or migrations.
- Production composition root or live facade source selection.
- Upload/processing writer integration.
- Authentication, authorization, tenant isolation, or rate limiting.
- Production CORS, gateway, TLS, telemetry, retention, or archive policy.
- Mutation endpoints.
- Raw encrypted blob storage.
- FlowSync Document Intelligence UI.
- OCR, LLM processing, or external services.
