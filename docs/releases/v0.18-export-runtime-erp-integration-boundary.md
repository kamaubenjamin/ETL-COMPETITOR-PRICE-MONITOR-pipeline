# v0.18 Export Runtime / ERP Integration Boundary

## Release Status

Implemented, verified, and closed pending owner tag.

## Capabilities Added

- Export runtime contracts, readiness model, payload contracts, deterministic builder/normalization/fingerprints, and idempotency policy.
- Repository protocols, in-memory attempt/result store, deterministic queries, optimistic transitions, and duplicate prevention.
- Injected `ExportRuntimeService`, safe commands/results, deterministic placeholder adapters, lifecycle decisions, and audit intents.
- Tenant-filtered safe export-history GET contracts.
- Disabled prepare/export POST contracts with fixed safe HTTP 503 responses.
- FlowSync read-only export readiness/history placeholder and disabled export action.

## Architecture

```text
FlowSync Export Placeholder
  -> GET-only / disabled export API boundary
  -> Export provider
  -> Export Runtime Service
  -> Payload builder + idempotency policy
  -> Attempt/result repositories
  -> Placeholder adapter port
  -> Audit intents + lifecycle decisions
```

API-to-runtime execution is not activated in v0.18. No real ERP adapter, external call, mutation delivery, durable export store, audit writer, or lifecycle writer is included.

## API Contracts

Read routes: document export history, export-attempt collection, and export-attempt detail. Prepare/export POST routes return HTTP 503, `mutation_not_enabled`, `Export execution is not enabled.`, and `activation: deferred`.
