# v0.6 Extraction & Transformation Capability Hardening

**Release state:** Implementation complete; tag pending full regression in a provisioned environment
**Recommended tag:** `v0.6-extraction-transformation-capability-hardening`

## Capabilities Added

- Versioned deterministic transformation plans and strict configuration errors.
- Real Workflow Runtime `transform` execution for DataFrames and lists of row dictionaries.
- Fixed operation registry, flat field mapping, named regex mapping, coercion, and scalar transforms.
- Data-level validation with seven rule types, severity, failure policies, bounded details, and complete counts.
- Stable single/multi-key sorting with explicit direction and null placement.
- Grouped and dataset-level count/sum/average/minimum/maximum aggregation.
- Implemented workflow stages: `validate_data`, `sort`, and `aggregate`.
- Authoritative stage catalog aligned with Workflow Validator and runtime registration.
- Privacy-safe stage metadata and deterministic end-to-end integration coverage.

## Compatibility Notes

- Existing `TransformationPipeline.apply(rules)` remains supported.
- Legacy rule names and filter compatibility remain unchanged.
- Existing public Workflow, Document, Entity, and Matching contracts are preserved.
- New contracts and stage types are additive.
- No new package dependency or runtime-boundary exemption was introduced.

## Known Non-Goals

- OCR or scanned-document support.
- LLM-based extraction, mapping, validation, or correction.
- UI, API, dashboards, or external monitoring vendors.
- Full MDM/golden-record lifecycle and survivorship.
- Dynamic plugins, arbitrary Python callbacks, or a general-purpose rule language.
- Streaming/distributed execution, pivoting, window functions, or time-series aggregation.
