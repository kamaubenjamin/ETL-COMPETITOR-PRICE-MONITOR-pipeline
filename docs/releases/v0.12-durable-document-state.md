# v0.12 Durable Document State

## Release Name

Durable Document State v1

## Capabilities Added

- Persistence configuration, schema metadata, safe errors, and migration/ledger contracts.
- File-backed standard-library SQLite connection and transaction layer.
- Explicit relational schema for all ten Document State record families plus migration ledger.
- Contract-driven SQLite mappers and durable read/write repositories.
- Ordered transactional migration application with SHA-256 checksum verification.
- Transaction-consistent count/page reads and atomic writes.
- Optimistic `expected_version` compare-and-swap updates.
- Append idempotency using stable keys, canonical content hashes, and unique constraints.
- Shared in-memory/SQLite conformance, durability, rollback, snapshot, and basic concurrency verification.
- Explicit `in_memory` or `sqlite` repository composition with no silent fallback.
- Deferred `future_postgres` backend identity without implementation or activation.

## Architecture

```text
Document Intelligence API
  -> FacadeDocumentIntelligenceProvider
  -> Workflow Query Facade
  -> DocumentStateQueryFacadeAdapter
  -> Document State repositories
  -> DocumentStateComposition
  -> SQLite durable backend OR in-memory backend
```

SQLite is approved for local/development durability only. Production activation is not part of this release.

## Compatibility Notes

This release is additive. v0.9 API paths, GET-only methods, payload meanings, envelopes, pagination, request IDs, and security headers remain unchanged. Streamlit `local_preview` and `api_preview` behavior remains unchanged. API, Streamlit, and Workflow Query Facade do not import persistence modules directly.

## Known Non-Goals

- PostgreSQL/Supabase implementation or production database provisioning.
- Production composition activation or upload/processing writer integration.
- Auth/authz, tenant isolation, rate limiting, production CORS/gateway/TLS, or mutation endpoints.
- Production telemetry, backup/recovery, retention/archive/legal hold, or purge operations.
- Raw encrypted blob storage.
- FlowSync Document Intelligence UI.
- OCR, LLM processing, or external services.
