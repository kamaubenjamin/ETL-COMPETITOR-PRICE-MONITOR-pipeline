# v0.16 Production Composition / Runtime Selection

## Release Name

Production Composition / Runtime Selection v1

## Capabilities Added

- Fixed runtime, backend, auth, identity-provider, API exposure, and Streamlit mode catalogs.
- Immutable runtime configuration with redacted serialization and safe validation errors.
- Explicit compatibility matrix and production fail-closed behavior.
- Outer composition root for Document State, lifecycle advancement, writer services, and Workflow Query Facade.
- App-scoped Document Intelligence API provider and auth composition.
- Non-authoritative Streamlit runtime preview labels.
- Boundary, provider-isolation, startup-failure, and privacy hardening.

## Runtime Architecture

```text
RuntimeConfig
  -> validate_runtime_config
  -> RuntimeComposition
  -> DocumentStateComposition
  -> LifecycleAdvancementService
  -> RuntimeWriterServices
  -> DocumentStateQueryFacadeAdapter
  -> WorkflowQueryFacadePort
  -> FacadeDocumentIntelligenceProvider
  -> Document Intelligence API
  -> Streamlit api_preview consumer
```

## Compatibility Notes

- Existing API paths, GET-only methods, payload meanings, envelopes, request IDs, pagination, and security headers are unchanged.
- Default local API app creation and compatibility provider remain available.
- Streamlit `local_preview` remains unchanged and default; `api_preview` labels are display-only.
- No endpoints, mutations, migrations, dependencies, or competitor-price behavior changed.

## Production Safety

Production remains deliberately unavailable. Current production combinations reject in-memory/SQLite persistence, disabled/local-demo/authenticated-placeholder auth, local identities, and deferred PostgreSQL. SQLite requires an explicit file path. Invalid configuration or composition never falls back to local defaults or compatibility providers.

Safe output excludes storage paths, identity-provider references, tokens, credentials, raw environment values, stack traces, and raw exceptions.

## Known Non-Goals

- PostgreSQL/Supabase persistence or production migrations.
- Real identity providers, secrets, token verification, sessions, MFA, or SSO.
- Production deployment infrastructure, diagnostics endpoint, telemetry, backup, or recovery.
- Public upload/mutation APIs, writer authorization, or child-record tenancy.
- FlowSync Document Intelligence UI/runtime integration or ERP/export integration.
