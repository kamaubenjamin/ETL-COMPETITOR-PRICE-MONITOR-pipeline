# v0.22 Deterministic Purchase-Order Demonstration

**Release class:** UAT / Technical Preview

**Data class:** Synthetic and non-confidential hosted data only

**Operation:** Read-only

**Status:** Owner closeout commit and annotated tag pending

## Summary

v0.22 adds a deterministic purchase-order demonstration to the protected FlowSync Documents experience. An authenticated owner in the bounded UAT tenant can list and filter the fictional `doc-002` record, open it through the normal document-detail route, inspect a canonical purchase-order result, review exact totals and two fictional line items, and see validation, matching, and lifecycle summaries.

## Highlights

- Canonical purchase-order schema with safe lineage, structured warnings, and validation findings.
- Machine-readable PDF classification and deterministic field/table extraction.
- Exact `Decimal` arithmetic and fixed-point API serialization.
- Fictional read-only hosted fixture; no controlled document content is deployed.
- Dedicated `GET /api/v1/documents/{document_id}/purchase-order` result endpoint.
- Existing list, type filter, detail, validation, matching, and lifecycle routes remain the navigation and authorization boundaries.
- Supabase tenant UUID authorization with a bounded UAT-only mapping from the authoritative tenant slug to the synthetic fixture namespace.
- Neutral presentation for intentionally absent upload-linked processing status.

## Security and Privacy

The controlled acceptance PDF remains ignored, private, and untracked. Hosted UAT contains only fictional data. Tokens, credentials, tenant UUIDs, raw text, local paths, and privileged configuration are not exposed by the demonstration. Request headers cannot select the synthetic fixture namespace, and unrelated tenants remain concealed.

## Hosted Acceptance

Owner verification at `ccca2af` confirmed authenticated session access, document listing, `purchase_order` filtering, normal detail navigation, the canonical PO panel, fictional parties and dates, exact totals, two line items, valid validation, and tenant isolation. The closeout adds only the local neutral handling for the optional upload-processing subresource; it requires owner review, commit, and deployment before hosted confirmation.

## Limitations

- Machine-readable PDFs and deterministic supported layouts only.
- No hosted upload, persistence, object storage, OCR, LLM extraction, queues, workers, approvals, exports, or document writes.
- Read-only free-tier UAT with synthetic/non-confidential data and no production SLA.
- Missing favicon and a non-blocking frontend bundle-size warning remain.

## Rollback

Use `ccca2af` as the last verified hosted application rollback point. Do not alter Supabase users, memberships, RLS, keys, CORS, or Vercel configuration as part of rollback.
